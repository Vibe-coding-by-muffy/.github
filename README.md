## 🇬🇧 English

### What is this?

This GitHub organization exists to prove a single point: **vibe coding can produce quality software**, as long as it's guided with intention and clarity.

I'm a developer who has always written every line of code myself. But I wanted to push a different question — not *"can AI help me code faster?"*, but rather:

> **"What can an AI build entirely on its own, from scratch, with no hand-holding?"**

Every project in this organization is generated and structured by AI. No manual patches. No silent rewrites. The process is transparent, the results are honest.

### The rules

- 🚫 No manual code corrections — AI output ships as-is
- 📋 Every project starts from a clear, detailed prompt
- 🔍 Results are evaluated honestly — failures included
- 📝 Each repo documents the process, not just the outcome

### Why does this matter?

Vibe coding has a reputation for producing sloppy, unmaintainable code. This org challenges that. The hypothesis: **the quality of AI-generated code is a direct reflection of the quality of the instructions it receives.**

### What to expect

- Real projects, not toy demos
- Honest post-mortems on what worked and what didn't
- A growing library of prompting patterns that actually produce solid code

## 🇫🇷 Français

### C'est quoi ce projet ?

Cette organisation GitHub a un seul objectif : **prouver que le vibe coding peut produire des logiciels de qualité**, à condition d'être guidé avec intention et précision.

Je code depuis toujours en écrivant moi-même chaque ligne. Mais j'ai voulu explorer une autre question — pas *"est-ce que l'IA peut m'aider à coder plus vite ?"*, mais plutôt :

> **"Qu'est-ce qu'une IA est capable de construire entièrement seule, de zéro, sans intervention humaine ?"**

Chaque projet de cette organisation est généré et structuré par une IA. Pas de corrections manuelles. Pas de réécriture silencieuse. Le processus est transparent, les résultats sont honnêtes.

### Les règles

- 🚫 Aucune correction manuelle du code — le résultat de l'IA est livré tel quel
- 📋 Chaque projet démarre d'un prompt clair et détaillé
- 🔍 Les résultats sont évalués honnêtement — y compris les échecs
- 📝 Chaque dépôt documente le processus, pas seulement le résultat final

### Pourquoi c'est important ?

Le vibe coding a la réputation de produire du code bâclé et difficile à maintenir. Cette organisation remet ce jugement en question. L'hypothèse : **la qualité du code généré par une IA est le reflet direct de la qualité des instructions qu'elle reçoit.**

### À quoi s'attendre

- De vrais projets, pas des démos jouets
- Des post-mortems honnêtes sur ce qui a fonctionné et ce qui a échoué
- Une bibliothèque croissante de patterns de prompting qui produisent réellement du code solide
